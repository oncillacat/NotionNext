A1 Objects & Assignment

In R, you store values in objects using the <- operator. Think of it as a labelled box holding a value.

Warning: Always use <-, not =. Using = inside function calls can cause subtle bugs.
R
a <- 34       ## we assign the number 34 to the object a
b <- 'character' ## we assign the word 'character' to the object b
a2 <- a * 2   ## multiply a times 2

# Print by typing the name
a
b
a2
# [1] 34 # [1] "character" # [1] 68

A2 Vectors

A vector is the fundamental R data structure — a sequence of values of the same type. Create one with c().

Warning: If you mix types inside c(), R silently coerces everything to the most general type — no error is shown.
R
absorbance    <- c(0.95, 0.80, 0.60, 0.43, 0.36, 0.21, 0.14, 0.08, 0.06)
concentration <- c(2000, 1500, 1000, 750, 500, 250, 125, 25, 0)
experiments   <- c('MTT', 'BCA', 'puromycin', 'ROS')
puromycin     <- c(20, 10, 5, 2, 1, 0.5, 0.2)

# Create a data frame from the vectors above
bsa_table <- data.frame(concentration, absorbance)
bsa_table

# What happens when you mix types?
mixed <- c(1, "hello", TRUE)
class(mixed)
# concentration absorbance # 1 2000 0.95 # 2 1500 0.80 # 3 1000 0.60 # 4 750 0.43 # 5 500 0.36 # 6 250 0.21 # 7 125 0.14 # 8 25 0.08 # 9 0 0.06 # [1] "character" ← everything coerced to character

A4 Functions & Arguments

Functions are "compacted scripts" that are already made for you to run lots of things at once. For example, you can calculate the square root or round numbers using built-in functions:

R — built-in functions
a <- 34
sqrt(a)          ## calculate the square root of a
round(3.141592)  ## round number (default: 0 decimal places)
# [1] 5.830952 # [1] 3

Arguments are the "conditions" of a function. For example, round() has an argument to set the number of decimal places — its default in R is 0.

You can inspect a function's arguments with args(), or read its full help page with ?:

R — inspecting arguments
args(round)  ## show the arguments of round()
# function (x, digits = 0) # NULL
R — changing the digits argument
# Specify by name
round(3.141592, digits = 2)

# R detects argument by position — name not required
round(3.141592, 2)

# You can reorder arguments if you name them
round(digits = 2, x = 3.141592)
# [1] 3.14 # [1] 3.14 # [1] 3.14
Best Practice: Put the non-optional arguments (the data you want the function to work on) first, then specify the names of all optional arguments. This makes your code much easier to read and understand.

A5 Subsetting

Use square brackets [] to extract elements — by position or by condition.

Interactive Subsetting

R — by position
experiments[2]  ## second element
# [1] "BCA"
R — by condition (single)
absorbance[absorbance > 0.5]  ## condition
# [1] 0.95 0.80 0.60
R — OR and AND conditions
absorbance[absorbance > 0.5 | absorbance < 0.1]  ## OR
absorbance[absorbance > 0.1 & absorbance < 0.5]  ## AND
# [1] 0.95 0.80 0.60 0.08 0.06 # [1] 0.43 0.36 0.21 0.14
R — %in% operator
experiments[experiments %in% c('BCA', 'ROS')]
# [1] "BCA" "ROS"

A6 Missing Data

R uses NA to represent missing values. NA is not zero — zero is a valid measurement.

Warning: Never use 0 for missing data. It corrupts your statistics silently.
R
concentration <- c(43.903, 47.871, NA, 48.456, 53.435)

mean(concentration)                       # NA is contagious
mean(concentration, na.rm = TRUE)          # correct way
concentration[!is.na(concentration)]       # logical negation
na.omit(concentration)                    # remove NAs
concentration[complete.cases(concentration)]  # same result
# [1] NA # [1] 48.41625 # [1] 43.903 47.871 48.456 53.435 # [1] 43.903 47.871 48.456 53.435 # [1] 43.903 47.871 48.456 53.435

A7 Factors

Factors represent categorical data with a fixed set of levels. By default levels are alphabetical.

R

# Convert back to character or to numeric codes
as.character(experiments_f)

B4 dplyr Functions

dplyr gives you a set of verb functions — just like in A4, each verb is a function with its own arguments. Click a verb below to see what it does to the data.

Tip: When using both filter() and select(), always filter first — then select() works on fewer rows.

mutate() — add or modify columns

Creates a new column from existing ones. The first argument is the data frame; subsequent arguments are new_col = expression.

R — mutate()
# Add a column with the row mean across all 14 replicates (cols 2–15)
experiment_info_mean <- mutate(experiment_info,
  average_abs = rowMeans(experiment_info[, 2:15]))
# Treatment A ... average_abs # MTT 0.95 ... 0.9321 # MTT 0.88 ... 0.8779 # BCA 0.80 ... 0.7950 # BCA 0.43 ... 0.4350 # ...
Exercise 1: Add two new columns in one mutate() call — the raw row mean and a rounded version:
R — Exercise 1
experiment_info_mean_round <- experiment_info %>%
  mutate(average_absorbance = rowMeans(experiment_info[, 2:15]),
         abs_round           = round(average_absorbance, 4))
# Treatment A ... average_absorbance abs_round # MTT 0.95 ... 0.93214286 0.9321 # MTT 0.88 ... 0.87785714 0.8779 # BCA 0.80 ... 0.79500000 0.7950 # BCA 0.43 ... 0.43500000 0.4350 # ...

select() — keep or drop columns

Keeps only the columns you name. Use -col to drop a column instead.

R — select()
# Keep only Treatment and first four replicates
select(experiment_info, Treatment, A, B, C, D)

# Drop a column with -
select(experiment_info, -N)

# Keep a range of columns
select(experiment_info, Treatment, A:D)
# Treatment A B C D # MTT 0.95 0.93 0.91 0.94 # MTT 0.88 0.87 0.90 0.86 # BCA 0.80 0.79 0.81 0.78 # ...

filter() — keep rows matching a condition

Keeps rows where the condition is TRUE. Multiple conditions are combined with & (AND) — or use | for OR.

R — filter()
# Single condition
filter(experiment_info, Treatment == "BCA")

# Multiple conditions (AND)
filter(experiment_info, Treatment == "BCA", A > 0.5)

# Match several values at once with %in%
filter(experiment_info, Treatment %in% c("BCA", "MTT"))
# Single condition → 2 rows (both BCA rows) # Multiple conditions → 1 row (BCA row where A > 0.5) # %in% → 4 rows (both BCA and both MTT rows)

arrange() — sort rows

Sorts the tibble by one or more columns. Wrap in desc() for descending order.

R — arrange()
# Sort by column A ascending (lowest first)
arrange(experiment_info, A)

# Sort descending (highest first)
arrange(experiment_info, desc(A))
# Ascending: Descending: # Treatment A Treatment A # ROS 0.08 MTT 0.95 # ROS 0.14 MTT 0.88 # puromycin 0.21 BCA 0.80 # ...

count() — tally rows per group

A quick shortcut: counts how many rows exist for each unique value of a column.

R — count()
count(experiment_info, Treatment)
# Treatment n # BCA 2 # MTT 2 # puromycin 2 # ROS 2

B5 The Pipe Operator

The pipe %>% passes the output of one function directly into the next — eliminating nested calls and intermediate variables.

Three ways to write the same pipeline

R
experiment_info %>%
  mutate(mean_ABCD = rowMeans(cbind(A, B, C, D), na.rm = TRUE)) %>%
  filter(Treatment == "BCA") %>%
  select(Treatment, mean_ABCD)
# Treatment mean_ABCD # BCA 0.795 # BCA 0.435

B6 Grouping & Summarising

group_by() — split data into groups

group_by() does not change the data itself — it adds invisible group labels so that the next function operates on each group separately. It is almost always paired with summarize().

R — group_by() alone changes nothing visible
# Before grouping
nrow(experiment_info)         ## still 8 rows

# After grouping — same 8 rows, but now labelled internally
grouped <- group_by(experiment_info, Treatment)
nrow(grouped)                  ## still 8 rows
groups(grouped)                ## shows the active grouping
# [1] 8 # [1] 8 # <list_of<sym>> # [[1]] # Treatment
Note: After group_by(), every subsequent dplyr verb runs per group until you call ungroup() to remove the grouping.

summarize() — collapse each group to one row

Reduces each group down to a single summary row. Arguments are new_col = summary_function(col). Common summary functions: mean(), sd(), median(), n(), min(), max().

R — group_by() + summarize()
experiment_info %>%
  group_by(Treatment) %>%
  summarize(
    n      = n(),                          ## count rows per group
    mean_A = mean(A, na.rm = TRUE),
    sd_A   = sd(A,   na.rm = TRUE)
  )
# Treatment n mean_A sd_A # BCA 2 0.615 0.255 # MTT 2 0.915 0.0495 # puromycin 2 0.285 0.106 # ROS 2 0.110 0.0424

Full pipeline: group_by() + summarize() + arrange()

Combining all three gives a clean ranked summary table in one readable pipeline.

R — full pipeline
experiment_info %>%
  group_by(Treatment) %>%
  summarize(mean_A = mean(A, na.rm = TRUE),
            sd_A   = sd(A,   na.rm = TRUE)) %>%
  arrange(desc(mean_A))          ## sort by highest mean first
# Treatment mean_A sd_A # MTT 0.915 0.0495 # BCA 0.615 0.255 # puromycin 0.285 0.106 # ROS 0.110 0.0424

B7 dplyr Pipeline → Linear Regression

This exercise ties together everything from Modules A and B: use pipes, mutate(), filter(), and select() to reduce the dataset, then fit a linear model to calculate protein concentrations.

Step 1 — Build experiment_info_reduced with a pipeline

Start from experiment_info. Use mutate() to compute a rounded row mean, filter() to keep only the BCA assay rows, and select() to retain just the columns you need.

R — pipes + mutate + filter + select
library(tidyverse)
experiment_info <- read_csv("puromycin.csv")

experiment_info_mean <- experiment_info %>%
  mutate(average_absorbance = rowMeans(experiment_info[, 2:15]),
         abs_round           = round(average_absorbance, 4)) %>%
  filter(Treatment == "BCA") %>%
  select(Treatment, abs_round)

experiment_info_mean
# A tibble: 2 × 2 # Treatment abs_round # <chr> <dbl> # 1 BCA 0.7950 # 2 BCA 0.4350

Step 2 — Fit a linear model with lm()

lm() fits a linear regression following the formula y = ax + b, where y is absorbance and x is concentration. The formula argument uses R's ~ notation: "absorbance as a function of concentration".

R — lm() on the BCA standard curve
# Standard curve data (known concentrations + measured absorbances)
bsa_table <- tibble(
  absorbance    = c(0.95, 0.80, 0.60, 0.43, 0.36, 0.21, 0.14, 0.08, 0.06),
  concentration = c(2000, 1500, 1000, 750, 500, 250, 125, 25, 0)
)

linear_model <- lm(formula = absorbance ~ concentration, data = bsa_table)
summary(linear_model)
# Call: # lm(formula = absorbance ~ concentration, data = bsa_table) # # Coefficients: # Estimate Std. Error t value Pr(>|t|) # (Intercept) 4.882e-02 2.441e-02 2.000 0.0859 # concentration 4.403e-04 2.226e-05 19.782 3.35e-07 *** # # R-squared: 0.9797 ← 97.97% of variance explained by the model

Step 3 — Extract the coefficients

coefficients() returns a named vector: [1] is the intercept (b), and [2] is the slope (a). We subset them with [] just like any other vector.

R — subsetting coefficients
coefficients(linear_model)          # see both coefficients

intercept     <- coefficients(linear_model)[1]  # b (intercept)
concentration <- coefficients(linear_model)[2]  # a (slope)
# (Intercept) concentration # 4.882e-02 4.403e-04

Step 4 — Calculate protein levels

Rearranging y = ax + b to solve for x (concentration) gives: x = (y − b) / a. We apply this to the abs_round column from our reduced dataset.

R — apply the formula
# x = (y - intercept) / slope
protein_levels <- (experiment_info_mean$abs_round - intercept) / concentration
protein_levels
# [1] 1706. 866.
What just happened? You used the BCA standard curve (known absorbance → known concentration) to build a model, then applied that model to your unknown samples — converting their absorbance readings into estimated protein concentrations (µg/mL).

C1 Linear Regression

Use lm(y ~ x) to fit a linear model. The ~ is R's formula notation: "y as a function of x".

R
absorbance    <- c(0.95, 0.8, 0.6, 0.43, 0.36, 0.21, 0.14, 0.08, 0.06)
concentration <- c(2000, 1500, 1000, 750, 500, 250, 125, 25, 0)

model <- lm(absorbance ~ concentration)
coef(model)

# Predict at concentration = 800
predict(model, newdata = data.frame(concentration = 800))
# (Intercept) concentration # 0.04882 0.00044 # [1] 0.4003 ← predicted absorbance at 800 µg/mL
